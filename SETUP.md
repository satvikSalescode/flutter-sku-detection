# SKU Detector – Setup Guide

On-device YOLOv8 inference with TFLite GPU delegate on Android.

---

## Prerequisites

| Tool | Version |
|------|---------|
| Flutter SDK | ≥ 3.3 |
| Dart SDK | ≥ 3.3 |
| Python | ≥ 3.9 |
| ultralytics | latest (`pip install ultralytics`) |
| Android device | API 24+, Vulkan or OpenGL ES 3.1 |

---

## Step 1 – Convert the model

```bash
cd /path/to/flutter-sku-detection

# Install ultralytics if needed
pip install ultralytics

# Run the converter (uses best.pt path baked into the script)
python convert_model.py
```

The script will:
- Export `best.pt` → TFLite (FP16 by default)
- Print the number of classes and class names
- Write `assets/labels.txt`
- Write `model_meta.json`

Copy the exported `.tflite` file into the `assets/` folder:

```bash
# The exported file is typically at:
#   /Users/satvikchaudhary/Desktop/IRED DOCS/All_Epochs/best_saved_model/best_float32.tflite
cp "/Users/satvikchaudhary/Desktop/IRED DOCS/All_Epochs/best_saved_model/best_float32.tflite" assets/
```

---

## Step 2 – Update constants in inference_service.dart

Open `lib/services/inference_service.dart` and update:

```dart
const String kModelAsset  = 'assets/best_float32.tflite'; // filename of your .tflite
const int    kNumClasses  = 1;   // ← number printed by convert_model.py
const int    kInputSize   = 640; // ← keep 640 unless you changed it
```

---

## Step 3 – Create the Flutter project

```bash
# Create a new Flutter project (only needed once)
flutter create sku_detector
cd sku_detector

# Copy the source files from this folder into the new project
cp -r /path/to/flutter-sku-detection/lib        ./
cp -r /path/to/flutter-sku-detection/assets     ./
cp    /path/to/flutter-sku-detection/pubspec.yaml ./
```

Or, if you want to use this folder directly as the Flutter project root,
just run `flutter create .` here (it won't overwrite existing files).

---

## Step 4 – Apply Android configuration

See `android_changes.md` for the exact edits needed in:
- `android/app/build.gradle`
- `android/app/src/main/AndroidManifest.xml`

---

## Step 5 – Install dependencies & run

```bash
flutter pub get
flutter run --release   # --release enables GPU optimisations
```

---

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `Failed to load model` | Make sure the .tflite filename matches `kModelAsset` |
| `GPU delegate unavailable` | Device doesn't support Vulkan; app auto-falls back to CPU |
| Many false positives | Lower `kConfThreshold` in inference_service.dart |
| Wrong class labels | Re-run `convert_model.py`, check labels.txt has correct names |
| Output shape mismatch | Check logcat for "Output tensor shape" and adjust decoding if needed |
